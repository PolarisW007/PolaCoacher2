# 讲解题图生成策略文档

> 版本：v2.0 | 更新日期：2026-03-14  
> 适用场景：AI讲解页场景图、文档封面图、分享配图

---

## 一、现状问题分析

### 1.1 当前方案缺陷

| 问题 | 现状 | 期望 |
|------|------|------|
| **风格单一** | 仅有"赛博朋克"和"吉卜力3D"两种风格 | 根据文档类型灵活匹配 |
| **内容关联弱** | 只靠关键词数量判断风格，不理解内容 | 图片能帮助读者理解当页内容 |
| **构图重复** | 同一文档所有页面构图趋同 | 构图类型轮转，画面多样 |
| **忽略已知IP** | 无法识别经典作品、影视游戏对应素材 | 对已知IP参考原作视觉风格 |
| **模型非最优** | 使用 `wanx2.1-t2i-turbo`（速度优先） | 升级为 `wanx2.1-t2i-plus`（质量优先） |

### 1.2 模型升级建议

| 模型 | 参数量 | FID得分 | 特点 | 推荐场景 |
|------|--------|---------|------|---------|
| `wanx2.1-t2i-turbo` | 8亿 | 15.2 | 速度快（2.7s/张），细节少 | 批量封面、低优先级场景 |
| **`wanx2.1-t2i-plus`** ✅ | **24亿** | **8.7** | 艺术细节丰富，光影层次佳 | **讲解场景图（主要模型）** |
| `wan2.6-t2i` | 最新版 | 待测 | 多模态能力更强 | 后续可测评引入 |

**结论：将讲解场景图的生成模型从 `wanx2.1-t2i-turbo` 升级为 `wanx2.1-t2i-plus`。**

## 二、文档类型分类体系

### 2.1 七大文档类型

每份文档在处理阶段（`doc_processor.py`）通过 AI 一次性判断类型，存储为文档级元数据，后续所有页面图片生成均参考此类型。

| 类型 ID | 类型名称 | 典型文档 | 判断关键词/特征 |
|---------|---------|---------|---------------|
| `academic` | 学术论文/技术报告 | AI论文、科研报告、白皮书 | 摘要、实验、方法、结论、论文、研究、模型 |
| `literature` | 文学/小说/诗歌 | 小说、散文、诗集、戏剧 | 人物姓名、情节描述、对白、文学性叙述 |
| `history` | 历史/传记/人文 | 历史书、人物传记、考古报告 | 朝代、历史人物、时间线、史料、考古 |
| `science_pop` | 科普/自然/医学 | 科普读物、自然百科、健康手册 | 动植物、科学现象、天文、生物、医学 |
| `philosophy` | 哲学/社科/心理 | 哲学著作、社会学、心理学书 | 思想、理论、观点、批判、意识、存在 |
| `tech_dev` | 编程/技术开发 | 技术文档、编程教程、架构设计 | 代码、API、函数、数据库、开发、部署 |
| `known_ip` | 已知IP作品 | 经典名著、热门影视/游戏对应材料 | 通过专项检测（见2.2节） |

### 2.2 已知IP检测

在文档处理时，额外向 Qwen 提问：  
*"这份文档的标题和内容是否对应某部知名的影视作品、游戏、图书IP？如果是，请返回IP名称和视觉风格描述；如果不是，返回null。"*

**已知IP的图片生成策略：**  
- 将IP的视觉风格关键词注入 Prompt（如：《哈利·波特》→ `magical British boarding school, warm candlelight, gothic stone corridors`）
- 让用户通过图片联想到原作，加深理解
- 不直接复制原作画面，而是"同世界观"的场景诠释

## 三、构图类型轮转策略

### 3.1 五种构图类型

为避免同一文档内页面图片过于相似，按页面序号轮转使用不同构图：

| 构图类型 | 英文标签 | 描述 | 适合内容 |
|---------|---------|------|---------|
| **全景场景** | `panoramic_scene` | 宏观环境，展现整体氛围 | 背景介绍页、总览页 |
| **焦点人物/对象** | `focal_subject` | 主体居中，背景虚化 | 人物介绍、核心概念页 |
| **流程/关系图示** | `process_diagram` | 箭头、节点、流程线 | 方法步骤、结构关系页 |
| **象征静物** | `symbolic_still` | 有意义的物品组合、微距细节 | 比喻类内容、抽象观点 |
| **对比/并列** | `split_comparison` | 左右对比或前后对比构图 | 对比分析、优缺点对比页 |

**轮转规则（页面索引 `slide_idx` 决定）：**
```
轮转顺序 = ["panoramic_scene", "focal_subject", "process_diagram", "symbolic_still", "split_comparison"]
当前构图 = 轮转顺序[slide_idx % 5]
```

特殊规则：
- 封面页（第0页）强制使用 `panoramic_scene`（展现整体世界观）
- 结论/总结页强制使用 `symbolic_still`（沉淀感，收束情绪）
- 如果当页讲解文本含有"对比""vs""区别"等词，优先使用 `split_comparison`

## 四、各文档类型 Prompt 生成模板

### 4.1 通用 Prompt 结构

每张图的最终 Prompt 由三个部分拼接组成：

```
[风格基底 Style Base]  +  [内容描述 Content Core]  +  [构图指令 Composition]  +  [质量修饰 Quality Tags]
```

**质量修饰（所有类型共用尾缀）：**
```
masterpiece, best quality, highly detailed, professional illustration, 
coherent composition, visually engaging, 1024x576, cinematic lighting
```

---

### 4.2 `academic` — 学术论文/技术报告

**视觉风格：** 清晰的教学图解风格，类似高质量教科书插图或 Nature/Science 期刊配图。干净明亮，重信息密度，轻装饰。

**风格基底：**
```
clean educational illustration style, textbook diagram aesthetic, 
clear white background or soft light gray, precise linework, 
scientific visualization, infographic quality, no unnecessary decoration
```

**构图要素：**
- `process_diagram` → 流程图/架构图，带标注箭头，节点清晰
- `focal_subject` → 核心概念的视觉化表达（如神经网络节点、分子结构）
- `panoramic_scene` → 研究场景（实验室、数据看板、学术演讲厅）

**Prompt 示例（ReAct 论文第4页 "核心思想"）：**
```
Clean educational diagram illustration, a human brain and robotic arm working together, 
reasoning thought-bubbles on the left showing logical chains, action execution nodes on the right 
with feedback arrows, connected by glowing synapse-like pathways, light blue and white palette, 
scientific journal illustration style, precise linework with soft shadows, 
textbook quality, informative and clear layout.
masterpiece, best quality, highly detailed, professional illustration
```

---

### 4.3 `literature` — 文学/小说/诗歌

**视觉风格：** 充满情感张力的叙事插画风格，类似精装版文学书籍内页插图，注重氛围与情感传达。

**风格基底：**
```
literary book illustration style, atmospheric narrative scene, 
painterly texture with visible brushstrokes, emotional depth, 
muted but rich color palette, storytelling composition
```

**构图要素：**
- `panoramic_scene` → 故事发生的世界/场景全貌
- `focal_subject` → 关键人物的特写或姿态
- `symbolic_still` → 故事中的标志性物件（如《了不起的盖茨比》的绿灯）

**已知IP示例（《百年孤独》第3页）：**
```
Literary illustration, a magical realist Latin American village scene bathed in golden afternoon light, 
ancient banana trees lining a dusty street, a solitary patriarch figure standing in a doorway, 
butterflies swirling around him, Gabriel García Márquez magical realism aesthetic, 
warm earth tones with occasional surreal impossible details, 
painterly brushstroke texture, melancholic yet beautiful atmosphere.
masterpiece, best quality, highly detailed, professional illustration
```

---

### 4.4 `history` — 历史/传记/人文

**视觉风格：** 历史感浓厚的插画风格，参考历史画作、古典油画或精细的历史纪录片视觉风格，年代感真实。

**风格基底（根据历史年代自动匹配）：**
```
# 中国古代：
traditional Chinese ink painting with color wash, meticulous gongbi style, 
period-accurate Tang/Song/Ming dynasty (根据时代) costumes and architecture

# 西方历史：
classical European oil painting style, Rembrandt-like chiaroscuro lighting, 
historical realism, museum quality illustration

# 近现代：
documentary photography aesthetic, sepia tone or period color grading,
archival illustration style
```

**Prompt 示例（秦朝统一历史文档第5页）：**
```
Classical Chinese historical painting, a grand bird's-eye view of the Qin dynasty capital Xianyang,
terracotta-colored palace walls, bronze chariots and armored soldiers in precise formations below,
the First Emperor overlooking from a high terrace, meticulous gongbi brushwork style,
muted earth red, jade green, and midnight black palette, 
ancient scroll painting texture, historically evocative atmosphere.
masterpiece, best quality, highly detailed, professional illustration
```

---

### 4.5 `science_pop` — 科普/自然/医学

**视觉风格：** 国家地理杂志、BBC纪录片截图、科学美国人级别的高清科普插图风格。自然真实，细节丰富，充满探索感。

**风格基底：**
```
National Geographic quality illustration, nature documentary aesthetic, 
photorealistic natural environment, scientifically accurate depiction,
vibrant natural colors, macro photography detail or aerial wildlife shot style
```

**Prompt 示例（黑洞科普第6页）：**
```
Scientific illustration of a supermassive black hole in deep space, 
photorealistic NASA Hubble-quality render, accretion disk glowing in orange and golden light, 
gravitational lensing distorting background stars, event horizon in perfect darkness,
accurate astrophysics representation, cosmic dust and star field background,
awe-inspiring and educational visual composition.
masterpiece, best quality, highly detailed, cinematic lighting
```

### 4.6 `philosophy` — 哲学/社科/心理

**视觉风格：** 超现实主义或象征主义绘画风格（参考马格利特、达利的视觉语言），用视觉隐喻表达抽象思想。

**风格基底：**
```
surrealist philosophical illustration, symbolic visual metaphor, 
René Magritte-inspired dreamlike realism, thought-provoking visual paradox,
deep conceptual imagery, muted sophisticated palette with accent highlights
```

**Prompt 示例（尼采"超人"哲学第7页）：**
```
Surrealist philosophical painting, a solitary human figure standing at the edge of a vast mountain cliff,
the figure gradually transforms into pure golden light ascending upward,
below lies conventional society depicted as a clockwork mechanical city,
René Magritte surrealist style with hyper-real textures,
deep indigo sky, golden transformation gradient, dramatic philosophical atmosphere,
symbolic and thought-provoking, museum quality oil painting aesthetic.
masterpiece, best quality, highly detailed, professional illustration
```

---

### 4.7 `tech_dev` — 编程/技术开发

**视觉风格：** 现代科技图解风格，扁平化+适度3D，类似高质量技术博客配图或GitHub README图表。清晰、专业，但比学术图更有设计感。

**风格基底：**
```
modern tech illustration, flat design with subtle 3D depth,
isometric or semi-isometric perspective, clean code-inspired aesthetics,
dark or light mode tech color palette (deep navy, electric blue, or clean white),
professional software engineering visual style
```

**Prompt 示例（微服务架构第8页）：**
```
Modern isometric tech illustration, a colorful microservices architecture diagram,
multiple service boxes connected by API gateway in the center,
data flows visualized as glowing connection lines between services,
clean flat design with subtle 3D isometric perspective,
dark navy background with neon accent colors (blue, teal, orange) for different services,
professional engineering diagram aesthetic, clear and informative.
masterpiece, best quality, highly detailed, professional illustration
```

### 4.8 `known_ip` — 已知IP作品

**视觉风格：** 在不侵权的前提下，采用与原IP相同的视觉风格语言，让用户一眼联想到原作，通过图片辅助理解原作内容。

## 五、完整处理流程

### 5.1 文档处理阶段（一次性）

```
文档上传
    ↓
[doc_processor.py] 文本提取 & 分段
    ↓
[ai_service.py] Qwen 分析文档类型 (classify_document_type)
    → 判断: academic / literature / history / science_pop / philosophy / tech_dev
    → 判断: 是否为已知IP (detect_known_ip)
    → 返回: { doc_type, ip_name, ip_visual_style }
    ↓
存储文档类型元数据到数据库 document.doc_type / document.ip_info
    ↓
生成讲解 slides + TTS 音频（现有逻辑）
    ↓
[异步] 逐页生成场景图（见5.2）
```

### 5.2 逐页场景图生成（异步批量）

```
读取 slide[i] 的 { title, points, lecture_text }
读取文档级 { doc_type, ip_info }
    ↓
[ai_service.py] determine_composition_type(slide_idx, lecture_text)
    → 默认轮转: ["panoramic", "focal", "process", "symbolic", "split"][i%5]
    → 特殊规则覆盖 (封面→panoramic, 对比内容→split)
    ↓
[ai_service.py] generate_slide_scene_prompt_v2(
    title, points, lecture_text,
    doc_type, ip_info, composition_type
)
    → 拼接: 风格基底 + AI生成的内容描述 + 构图指令 + 质量标签
    → 调用 Qwen Plus 生成英文 Prompt（max 150词）
    ↓
[ai_service.py] _call_wanx_image(
    prompt=生成的英文Prompt,
    model="wanx2.1-t2i-plus",   ← 升级为Plus模型
    size="1024*576"
)
    ↓
下载保存到 data/slide_images/doc_{id}_slide_{idx}.png
更新数据库 slide.scene_image_url
    ↓
前端轮询获取图片URL并展示
```

### 5.3 封面图生成（文档封面）

```
文档处理完成
    ↓
读取 { doc_type, ip_info, title, summary, key_points }
    ↓
generate_cover_image_prompt_v2(doc_type, ip_info, title, summary, key_points)
    → 封面使用 panoramic_scene 构图
    → 融合文档类型风格基底
    → 尺寸: 1024*1024 (正方形，适配封面展示)
    ↓
_call_wanx_image(model="wanx2.1-t2i-plus", size="1024*1024")
```

## 六、Qwen Prompt 生成的 Meta-Prompt 模板

以下是调用 Qwen 生成画图 prompt 时的 system 和 user prompt 设计：

### 6.1 System Prompt（统一使用）

```
You are an expert AI art director and prompt engineer for educational illustration. 
Your job is to create detailed English image generation prompts that help readers 
visually understand the document content. 

Rules:
1. Output ONLY the English prompt, no explanations or prefixes
2. Maximum 150 words
3. The image must be educational and content-relevant, not merely decorative
4. Follow the specified document type style and composition type exactly
5. Include specific visual elements that directly relate to the content being explained
```

### 6.2 User Prompt 模板

```python
def build_meta_prompt(
    doc_type: str,
    doc_type_style_base: str,   # 从上方各类型风格基底查表
    ip_info: dict | None,
    composition_type: str,
    composition_desc: str,      # 构图描述
    slide_title: str,
    slide_points: list[str],
    lecture_text: str,
) -> str:
    ip_block = ""
    if ip_info:
        ip_block = f"\nThis content is from the known IP work: {ip_info['name']}. "\
                   f"Visual style reference: {ip_info['visual_style']}"
    
    return f"""
Document type: {doc_type}
Style base: {doc_type_style_base}
{ip_block}
Composition type: {composition_type} — {composition_desc}

Page title: {slide_title}
Key points: {'; '.join(slide_points[:4])}
Lecture context: {lecture_text[:300]}

Create an illustration prompt that:
1. Uses the specified style base as the visual foundation
2. Visually represents the key points on this page
3. Follows the {composition_type} composition type
4. Helps readers understand "{slide_title}" more intuitively
5. References specific visual elements mentioned in the content (not generic imagery)
"""
```

## 八、效果预期与注意事项

### 8.1 效果预期

| 文档类型 | 生成效果描述 |
|---------|------------|
| AI论文（如ReAct） | 清晰的方法流程图、对比示意图，帮助理解技术原理 |
| 小说（如《活着》） | 充满情感的叙事插图，与小说情节强烈共鸣 |
| 历史文档（如秦史） | 历史画风的场景复原，增强历史代入感 |
| 科普（如宇宙学） | 高清宇宙/自然可视化，媲美国家地理配图 |
| 技术开发文档 | 清晰的系统架构可视化，工程师一目了然 |
| 已知IP（如三体） | "同世界观"的场景插图，强化读者联想 |

### 8.2 注意事项

1. **成本控制**：`wanx2.1-t2i-plus` 比 `turbo` 生成时间约慢2.5倍（6.8s vs 2.7s/张），建议保持现有的异步后台生成策略，不阻塞用户体验。

2. **IP版权边界**：对已知IP的图片生成，只参考视觉风格语言，不生成具体的版权人物或场景，避免版权风险。

3. **文档类型识别容错**：若文档类型识别失败或不确定，默认降级为 `science_pop`（科普风格，通用性最强）。

4. **Prompt 字数限制**：`wanx2.1-t2i-plus` 中文提示词上限约500字，英文约150词，保持当前英文Prompt策略。

5. **渐进式迁移**：建议先对新上传文档应用 v2 策略，历史文档保持现有图片不重新生成。
