## 一、产品概述

### 1.1 定位
书架是秘塔AI搜索平台的个人文档管理中心，Slogan 为 **"重构您的阅读体验"**。它为用户提供文档的收藏、管理、分类和AI增强阅读能力，是平台从"搜索"延伸到"知识管理"的核心模块。

### 1.2 入口
- 左侧固定导航栏 > "书架"（带书本图标）
- URL路径：`/bookshelf`
- 面包屑导航：首页 > 书架

### 1.3 使用前提
- 需要用户登录（未登录状态下显示空书架框架，操作时触发登录拦截）
- 登录方式：微信扫码登录、手机验证码登录、账号密码登录

## 三、核心功能模块

### 3.1 Tab视图切换（四种模式）

书架提供4个Tab标签进行视图切换，选中标签以蓝色下划线高亮显示：

#### 3.1.1 默认模式
- **功能**：以卡片网格展示用户书架中的所有文档
- **空状态**：显示一个虚线框卡片，内含"+"图标和文字"添加文档到书架"
- **有内容时**：以书本封面卡片形式展示文档列表
- **操作入口**："添加文档到书架"按钮

#### 3.1.2 讲解模式
- **功能**：筛选展示可进行AI讲解的文档
- **空状态**：虚线框卡片显示"上传文档来讲解"
- **核心能力**：对文档内容进行AI驱动的语音/文字讲解，帮助用户理解复杂内容
- **操作入口**："上传文档来讲解"按钮

#### 3.1.3 文档模式
- **功能**：以纯文档列表视图展示文件
- **空状态**：虚线框卡片显示"添加文档到书架"
- **侧重**：更偏向文件管理视角，展示文件名、类型、大小等元信息

#### 3.1.4 分组模式
- **功能**：按用户自定义分组管理文档
- **空状态**：虚线框卡片显示"创建分组"
- **操作入口**："创建分组"按钮，支持用户建立文件夹/标签式的分类体系

### 3.2 文档添加

#### 3.2.1 添加文档到书架
- **触发方式**：点击空状态卡片或右上角"+"按钮
- **预期弹窗**：弹出文档来源选择弹窗，提供以下方式：
  - 从文库搜索添加
  - 上传本地文档
  - 从URL导入（推测）

#### 3.2.2 去文库搜索
- **功能**：跳转至秘塔文库搜索页面
- **流程**：搜索文库中的已有文档 → 点击收藏/添加到书架
- **搜索范围**：全网文档、学术论文、PDF文档等

#### 3.2.3 上传本地文档
- **功能**：从用户设备上传文档文件
- **支持格式**（基于平台能力推断）：PDF、Word、TXT、Markdown等
- **上传后处理**：文档解析 → AI索引 → 加入书架

### 3.3 分组管理

- **创建分组**：在分组模式下，点击"创建分组"按钮
- **分组操作**：
  - 创建/重命名/删除分组
  - 将文档拖入或添加到分组
  - 分组排序

### 3.4 文档阅读与AI增强

当书架中有文档时，点击文档卡片进入文档详情页，预期包含：
- **文档阅读**：PDF/文本在线阅读
- **AI讲解**：对文档内容进行AI驱动的讲解
- **AI问答**：基于文档内容的对话式问答
- **笔记标注**：对文档关键段落进行标注和笔记
- **分享**：分享文档或阅读笔记

## 四、请求与响应机制分析

### 4.1 核心API列表

| API端点 | 方法 | 说明 | 认证 |
|---------|------|------|------|
| `/api/my-info` | GET | 获取当前登录用户信息 | 需要登录（401拦截） |
| `/api/recently-used-topic` | GET | 获取最近使用的专题 | 需要登录 |
| `/api/metaso-ai-config` | GET | 获取平台全局配置（首页Demo等） | 无需登录 |
| `/api/presentation/config` | GET | 获取PPT/演示配置（页数、内容密度、语言、主题） | 无需登录 |
| `/api/chapter/setting` | GET | 获取章节阅读设置 | 无需登录 |
| `/api/search-template` | GET | 获取搜索模板（歌曲生成、自定义技能等） | 无需登录 |
| `/api/meta-config/by-key?configKey=scholarScope` | GET | 获取学术搜索范围配置 | 无需登录 |
| `/admin/user/event_report` | POST | 用户行为埋点上报 | 无需登录 |
| `files.metaso.cn/api/official-website` | GET | 获取官方文档/网站信息 | 无需登录 |
| `files.metaso.cn/api/ppt/config` | GET | PPT功能配置 | 无需登录 |

### 4.2 事件埋点体系

平台采用双通道埋点：
1. **内部埋点**：`POST /admin/user/event_report`
2. **第三方埋点**：UserMaven (`events.usermaven.com/api/v1/event`)

#### 关键埋点事件：

| 事件名 | 触发时机 | 数据示例 |
|--------|---------|---------|
| `browse-page` | 页面浏览 | `{"pageName":"Bookshelf","pathname":"/bookshelf"}` |
| `bookshelf` | 进入书架页面 | `"1"` (计数) |
| `show-login-dialog` | 显示登录弹窗 | `{"desc":"left-menu-login-btn"}` |
| `login-way` | 切换登录方式 | `"wxQRCode"` / `"account"` |
| `login-form-submit` | 提交登录表单 | `{"client":"desktop"}` |
| `my-info-401` | 用户信息获取失败(未登录) | `{"client":"desktop"}` |
| `logout` | 登出/未登录状态 | `{"desc":"global-effect-myinfo-error"}` |
| `pageview` | 页面访问(UserMaven) | URL、来源、标题等 |
| `$pageleave` | 页面离开(UserMaven) | URL、停留时间等 |

### 4.3 认证机制

- **认证方式**：基于Cookie/Session的认证
- **认证校验**：通过 `GET /api/my-info` 接口校验登录状态
- **未登录响应**：`{"errCode": 401, "errMsg": "需要登录"}`
- **登录状态传播**：登录成功后通过Cookie在全站传递认证信息
- **会话标识**：使用 `local_sid` 和 `tid`（tracking ID）追踪用户会话

### 4.4 响应数据格式

统一JSON响应格式：
```json
{
  "errCode": 0,        // 0=成功，401=需要登录，其他=错误
  "errMsg": "success",  // 错误描述
  "data": {}            // 业务数据
}
```

事件上报响应格式：
```json
{
  "code": 0,
  "msg": "success"
}
```

## 六、技术架构分析

### 6.1 前端技术栈
- **框架**：Next.js (App Router, RSC)
- **UI库**：Material-UI (MUI)
- **路由**：`/bookshelf` 独立页面，支持 `search/[id]` 动态路由
- **状态管理**：React Context/Hooks
- **部署**：SSR + 客户端混合渲染

### 6.2 后端服务
- **主服务**：`metaso.cn`（API + 页面渲染）
- **文件服务**：`files.metaso.cn`（文件上传/PPT/文档相关）
- **CDN**：`cdn.metaso.cn` / `metaso-static.oss-accelerate.aliyuncs.com`（静态资源）
- **埋点服务**：`events.usermaven.com`（第三方UserMaven）

### 6.3 域名结构
| 域名 | 用途 |
|------|------|
| `metaso.cn` | 主站（页面 + API） |
| `files.metaso.cn` | 文件服务（上传、PPT、文档） |
| `cdn.metaso.cn` | CDN静态资源 |
| `static-1.metaso.cn` | 静态资源（Next.js构建产物） |
| `events.usermaven.com` | 第三方行为分析 |

## 七、交互流程图

### 7.1 书架主流程
```
用户进入书架页面
     │
     ├─── 未登录 ──→ 展示空书架框架 ──→ 点击操作 ──→ 弹出登录弹窗
     │                                                    │
     │                                              登录成功
     │                                                    │
     └─── 已登录 ──→ 加载书架数据 ──→ 展示文档卡片列表

                          │
                          ├── 添加文档 ──→ 文库搜索/本地上传
                          │
                          ├── 分组管理 ──→ 创建/编辑/删除分组
                          │
                          └── 点击文档 ──→ 进入阅读/讲解页面

```
点击"添加文档到书架"
     │
     ├── 去文库搜索 ──→ 搜索页面 ──→ 选择文档 ──→ 添加到书架
     │
     ├── 上传本地文档 ──→ 文件选择器 ──→ 上传 ──→ 解析 ──→ 入库
     │
     └── 从URL导入（推测）──→ 输入URL ──→ 抓取 ──→ 解析 ──→ 入库

## 九、竞品差异化特征

1. **AI讲解模式**：独有的文档AI讲解功能，不仅是阅读，而是"听课式"学习
2. **搜索+书架联动**：文库搜索结果可直接收藏到书架，形成搜索→收藏→阅读闭环
3. **多模式视图**：4种视图模式满足不同场景（浏览/学习/管理/分类）
4. **无广告体验**：强调"没有广告，直达结果"

---

## 十、总结

秘塔AI搜索的书架模块是一个将**文档管理**与**AI增强阅读**结合的知识管理工具。其核心价值主张是"重构阅读体验"，通过四种视图模式（默认、讲解、文档、分组）满足用户从浏览到深度学习的不同需求。结合平台强大的搜索能力和文件处理能力，书架模块形成了完整的"搜索 → 收藏 → 管理 → 阅读 → 学习"知识管理闭环。
